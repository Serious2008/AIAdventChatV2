## 1️⃣ Глобальный конфиг

### Что это?
Настройки Claude Code, которые применяются ко **всем проектам** на вашем компьютере.

### Где находится?
`~/.claude/config.json` (в домашней директории)

### Что можно настроить?
- Общие предпочтения
- Модель по умолчанию
- Глобальные разрешения
- Стиль вывода

### Пример:
```json
{
  "model": "sonnet",
  "permissions": {
    "defaultMode": "default"
  },
  "preferences": {
    "language": "ru"
  }
}
```

### Как использовать?
Глобальный конфиг загружается автоматически. Локальные настройки переопределяют глобальные.

## 2️⃣ Локальный конфиг

### Что это?
Настройки для **конкретного проекта** AIAdventChatV2.

### Где находится?
`.claude/settings.local.json` ✅ УЖЕ НАСТРОЕН

### Текущие настройки:
```json
{
  "permissions": {
    "allow": [
      "Bash(ls:*)",
      "Bash(cat:*)"
    ]
  },
  "model": "sonnet",
  "env": {
    "PROJECT_RULES": "CLAUDE.md",
    "PROJECT_NAME": "AIAdventChatV2",
    "PROJECT_ARCHITECTURE": "MVVM"
  }
}
```

### Что настроено?
- ✅ **permissions.allow** - разрешенные команды без подтверждения
- ✅ **model** - используемая модель (sonnet)
- ✅ **env** - переменные окружения с правилами проекта

### Как Claude использует это?
Claude автоматически знает:
- Правила проекта находятся в `CLAUDE.md`
- Проект называется `AIAdventChatV2`
- Архитектура - `MVVM`

## 3️⃣ Субагенты

### Что это?
Специализированные AI-агенты для конкретных задач. Они запускаются параллельно и работают автономно.

### Доступные субагенты:

#### 🔍 **Explore** - Исследование кодбазы
```
Запусти субагента Explore для поиска всех использований ClaudeService
```

Claude автоматически:
- Ищет файлы
- Анализирует зависимости
- Возвращает структурированный отчет

#### 📋 **Plan** - Планирование задач
```
Используй субагента Plan для планирования новой фичи RAG
```

Claude:
- Анализирует требования
- Разбивает на подзадачи
- Предлагает архитектуру

#### 👨‍💻 **Code Review** (можно создать кастомного)
```
Запусти code review агента для Services/ClaudeService.swift
```

### Как использовать субагентов?

**Вариант 1: Явное указание**
```
Запусти субагента Explore для анализа структуры Services/
```

**Вариант 2: Claude сам решает**
```
Найди все места где используется RAGService
```
(Claude автоматически запустит Explore агента)

**Вариант 3: Параллельные агенты**
```
Запусти 2 агента параллельно:
1. Explore для поиска моделей
2. Plan для архитектуры новой фичи
```

### Когда использовать субагентов?
- ✅ Сложный поиск в кодбазе
- ✅ Анализ архитектуры
- ✅ Планирование больших задач
- ✅ Когда нужно параллельно выполнить несколько задач

## 4️⃣ Скиллы (Slash-команды)

### Что это?
Переиспользуемые команды в `.claude/commands/`. Автоматизируют частые задачи.

### Доступные команды:

#### 📁 `/analyze-project`
Анализирует структуру проекта и ищет баги.

**Использование:**
```
/analyze-project
```

**Что делает:**
- Находит все Swift файлы
- Строит структуру проекта
- Ищет force unwrap, try!, as!
- Проверяет retain cycles
- Создает отчет

---

#### 📁 `/analyze-any-project`
Анализирует любой проект в текущей директории.

**Использование:**
```
/analyze-any-project
```

---

#### ⚙️ `/create-service`
Создает новый Service по шаблону проекта.

**Использование:**
```
/create-service
```

**Что делает:**
1. Читает CLAUDE.md
2. Спрашивает название и назначение
3. Создает файл в `Services/`
4. Использует правильный шаблон с MARK:, emoji, DI
5. Напоминает добавить в ChatViewModel

**Пример диалога:**
```
Вы: /create-service
Claude: Название сервиса? (например: Weather)
Вы: Notification
Claude: Назначение сервиса?
Вы: Управление push-уведомлениями
Claude: [создает NotificationService.swift]
```

---

#### 📦 `/create-model`
Создает новую Model по шаблону проекта.

**Использование:**
```
/create-model
```

**Что делает:**
1. Читает CLAUDE.md
2. Спрашивает название и поля
3. Создает файл в `Models/`
4. Добавляет Identifiable + Codable
5. Создает convenience и full initializers

**Пример:**
```
Вы: /create-model
Claude: Название модели?
Вы: NotificationSettings
Claude: Основные поля?
Вы: enabled: Bool, soundEnabled: Bool, badgeCount: Int
Claude: [создает NotificationSettings.swift]
```

---

#### 🔍 `/review-code`
Проверяет код на соответствие правилам проекта.

**Использование:**
```
/review-code
```

**Что делает:**
1. Читает CLAUDE.md
2. Спрашивает путь к файлу
3. Проверяет naming conventions
4. Ищет антипаттерны
5. Проверяет структуру
6. Создает детальный отчет с оценкой

**Пример:**
```
Вы: /review-code
Claude: Путь к файлу?
Вы: AIAdventChatV2/Services/ClaudeService.swift
Claude: [создает отчет с ✅/⚠️/❌]
```

---

### Как создать свою slash-команду?

1. Создайте файл `.claude/commands/my-command.md`
2. Добавьте frontmatter с описанием:
   ```markdown
   ---
   description: Описание команды
   ---

   Инструкции для Claude...
   ```
3. Используйте: `/my-command`

## 5️⃣ Примеры использования

### Пример 1: Создание нового функционала с нуля

**Задача:** Добавить систему уведомлений

**Шаги:**
```bash

/create-model

# 3. Создать сервис
/create-service

/review-code
# Вводим: AIAdventChatV2/Services/NotificationService.swift

# 5. Проанализировать проект
/analyze-project
```

```bash

/review-code

# 3. Исправить проблемы
Исправь найденные антипаттерны в ClaudeService.swift
```

```bash
Запусти 3 субагента параллельно:

```

## 🎯 Рекомендации

### Когда использовать что?

| Задача | Инструмент |
|--------|-----------|
| Настройка проекта | Локальный конфиг |
| Создание Service | `/create-service` |
| Создание Model | `/create-model` |
| Code review | `/review-code` |
| Анализ структуры | `/analyze-project` |
| Поиск в коде | Субагент Explore |
| Планирование | Субагент Plan |
| Комплексная задача | Комбинация субагентов |

---

### Best Practices

1. **Всегда читайте CLAUDE.md** перед началом работы
2. **Используйте slash-команды** для стандартных задач
3. **Запускайте субагентов параллельно** когда возможно
4. **Делайте code review** перед коммитом
5. **Обновляйте CLAUDE.md** при изменении правил
