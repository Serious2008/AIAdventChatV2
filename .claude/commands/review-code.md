ВАЖНО: Сначала прочитай файл CLAUDE.md для понимания правил проекта!

ЗАДАЧА: Провести code review файла на соответствие правилам AIAdventChatV2

КОНТЕКСТ ПРОЕКТА:
- Проект: macOS приложение на Swift/SwiftUI
- Архитектура: MVVM
- Правила проекта: CLAUDE.md
- Антипаттерны: см. раздел "Антипаттерны" в CLAUDE.md

ЗАПРОСИ У ПОЛЬЗОВАТЕЛЯ:
Путь к файлу для review (например: AIAdventChatV2/Services/ClaudeService.swift)

ЗАТЕМ АВТОМАТИЧЕСКИ:

**ШАГ 1: Прочитай CLAUDE.md**
Изучи разделы:
- Naming Conventions
- Паттерны и Best Practices
- Антипаттерны (что ЗАПРЕЩЕНО)
- Примеры хорошего кода

**ШАГ 2: Прочитай файл для review**
Используй Read tool для чтения файла

**ШАГ 3: Проверь Naming Conventions**
Проверь:
- Правильное ли имя файла?
- Соответствует ли паттернам (*Service, *ViewModel, *View)?
- camelCase для переменных?
- PascalCase для типов?

**ШАГ 4: Проверь на антипаттерны**
Найди:
- ❌ Force unwrap (!) без необходимости
- ❌ Force casting (as!)
- ❌ Force try (try!)
- ❌ Использование Any
- ❌ Хардкод API ключей
- ❌ print() без emoji или смысла
- ❌ Блокирование главного потока
- ❌ UI логика в ViewModel
- ❌ Прямое использование UserDefaults

**ШАГ 5: Проверь структуру файла**
- ✅ Есть ли заголовок с Created by?
- ✅ Правильные ли импорты?
- ✅ Есть ли MARK: комментарии?
- ✅ Правильная ли группировка кода?

**ШАГ 6: Проверь специфику типа**

Для Services:
- ✅ Dependency Injection через init?
- ✅ Async/await для асинхронных операций?
- ✅ Error handling (Result или throws)?
- ✅ Thread-safe операции?
- ✅ Emoji в логировании?

Для Models:
- ✅ Identifiable + Codable?
- ✅ let для неизменяемых полей?
- ✅ Два initializer'а?
- ✅ Computed properties?

Для Views:
- ✅ @ObservedObject для ViewModels?
- ✅ @State для локального состояния?
- ✅ Разбиение на computed properties?
- ✅ Нет бизнес-логики в View?
